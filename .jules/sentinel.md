## 2025-04-28 - Secure Error Handling & Input Validation in API Endpoints
**Vulnerability:** Information Exposure (leaking raw `error.message` to clients in 500 responses) and potential Application DoS via exceptions (missing type validation for strings before using `Buffer.from`).
**Learning:** Node.js Express endpoints without explicit type checking for parameters like `audioBase64` or `text` can crash or behave unexpectedly when passed malicious types (e.g., objects or arrays). Additionally, blindly passing `error.message` to `res.status(500).json()` can expose sensitive internal stack traces or configuration details if an unexpected error occurs.
**Prevention:** Always explicitly validate parameter types (e.g., `typeof param === 'string'`) before processing them, especially when passing them to native methods like `Buffer.from`. In `catch` blocks, always use generic fallback messages for client responses and rely on `console.error` for internal logging.
