
## 2025-04-27 - Prevent DoS via Buffer.from Exceptions and Error Leakage
**Vulnerability:** Passing an unvalidated request payload property (like `req.body.audioBase64`) directly to native API functions like `Buffer.from(payload, 'base64')` can cause a `TypeError` if the client sends an object or an array. Because the Node backend did not safely validate `typeof string`, this exception would crash the current execution flow. Furthermore, catching these errors and directly passing `error.message` to `res.status(500).json({ error: error.message })` exposes internal API workings and potential stack traces to the public.
**Learning:** Native functions that strictly expect strings (like `Buffer.from`) must always be protected with `typeof variable === 'string'` checks before execution when dealing with unvalidated `req.body` inputs. Additionally, generic error handlers must never return the raw exception message strings.
**Prevention:** Always explicitly validate the correct primitive type of request body parameters before using them in native Node methods, and enforce generic failure messages in `catch` blocks (e.g. "Internal server error") rather than relying on the caught exception's native message.
